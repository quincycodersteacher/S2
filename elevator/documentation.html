Elevator Simulation Project Documentation

This documentation provides a comprehensive summary of the classes and methods in the elevator simulation project. It is designed to allow a student to understand the functionality and recreate the project independently. The project simulates an elevator system in a building, including people moving between floors, elevator logic, and a visual representation on a canvas.

The project consists of three main files:

  • elevator.html: The HTML file that sets up the canvas and runs the simulation.
  • js/elevator.js: Contains the core classes for the simulation (Person, Elevator, Floor, Building, Simulation).
  • js/queue.js: Contains queue-related classes (Queue, FloorQueue, ElevatorQueue).

Below is a detailed breakdown of each class, including its purpose, properties, and methods. The classes are presented in the recommended build order to minimize dependencies and build complexity incrementally.

Step-by-Step Class Implementation

To recreate the project, implement the classes in the following order based on their dependencies. Each class includes an explanation of why it should be built at that step and what comes next.

1. Person Class

Why build this first: The Person class has no dependencies and represents the fundamental entity (people) that will be managed throughout the simulation. It's a simple data holder that other classes will reference.

Next: After Person, build the Queue class, as queues are needed to manage collections of people.

Represents an individual person in the simulation.

Properties:

  • name: A string identifier for the person (e.g., "a", "b").
  • originFloor: The floor number where the person starts.
  • destinationFloor: The floor number where the person wants to go.

Methods:

  • constructor(name, originFloor, destinationFloor): Initializes a new Person with the given name, origin floor, and destination floor.
  • equals(otherPerson): Checks if this person is equal to another based on name. Returns a boolean.
Testing Ideas
  • Test constructor: let p = new Person('a', 0, 2); console.assert(p.name === 'a' && p.originFloor === 0 && p.destinationFloor === 2);
  • Test equals: let p1 = new Person('a', 0, 1); let p2 = new Person('a', 1, 2); console.assert(p1.equals(p2) === true); let p3 = new Person('b', 0, 1); console.assert(p1.equals(p3) === false);

2. Queue Class

Why build this next: Queues are fundamental data structures used to manage ordered collections of people in floors and elevators. Build this after Person since it will hold Person instances.

Next: Build FloorQueue, which extends Queue for floor-specific logic.

A basic queue data structure for managing people.

Properties:

  • people: An array holding the queued elements.

Methods:

  • constructor(): Initializes an empty queue.
  • enqueue(element): Adds an element to the end of the queue.
  • dequeue(): Removes and returns the first element, or null if empty.
  • peek(): Returns the first element without removing it, or null if empty.
  • isEmpty(): Returns true if the queue has no elements.
  • size(): Returns the number of elements in the queue.
Testing Ideas
  • Test constructor and isEmpty: let q = new Queue(); console.assert(q.isEmpty() === true && q.size() === 0);
  • Test enqueue and dequeue: q.enqueue('a'); q.enqueue('b'); console.assert(q.size() === 2 && q.dequeue() === 'a' && q.size() === 1);
  • Test peek: console.assert(q.peek() === 'b' && q.size() === 1);

3. FloorQueue Class

Why build this next: FloorQueue extends Queue to provide floor-specific dequeue methods for handling people based on their destination relative to the floor. It depends on Queue.

Next: Build ElevatorQueue, which also extends Queue for elevator-specific logic.

A specialized queue for floors, extending Queue.

Methods:

  • Inherits constructor from Queue.
  • dequeueNextWithDestinationAbove(floor): Removes and returns the first person whose destination is above the given floor, or null if none.
  • dequeueNextWithDestinationBelow(floor): Removes and returns the first person whose destination is below the given floor, or null if none.
Testing Ideas
  • Test dequeueNextWithDestinationAbove: let fq = new FloorQueue(); let p1 = new Person('a', 0, 2); let p2 = new Person('b', 0, 1); fq.enqueue(p1); fq.enqueue(p2); console.assert(fq.dequeueNextWithDestinationAbove(0) === p1 && fq.size() === 1);
  • Test dequeueNextWithDestinationBelow: console.assert(fq.dequeueNextWithDestinationBelow(0) === p2);

4. ElevatorQueue Class

Why build this next: ElevatorQueue extends Queue to dequeue people based on their exact destination floor, used in elevators. It depends on Queue.

Next: Define the DIRECTION constants, as they are used in the Elevator class.

A specialized queue for elevators, extending Queue.

Methods:

  • Inherits constructor from Queue.
  • dequeue(floor): Removes and returns the first person whose destination matches the given floor, or null if none.
Testing Ideas
  • Test dequeue by floor: let eq = new ElevatorQueue(); let p1 = new Person('a', 0, 2); let p2 = new Person('b', 0, 1); eq.enqueue(p1); eq.enqueue(p2); console.assert(eq.dequeue(1) === p2 && eq.size() === 1);

5. Constants

Why build this next: The DIRECTION constants define UP and DOWN values used in elevator movement logic. Define these before Elevator since it references them.

Next: Build the Floor class, which uses FloorQueue.

  • DIRECTION: An object with UP (100) and DOWN (200) constants for elevator direction.
Testing Ideas
  • Test constants: console.assert(DIRECTION.UP === 100 && DIRECTION.DOWN === 200);

6. Floor Class

Why build this next: The Floor class represents individual floors in the building and uses FloorQueue to manage waiting people. It depends on FloorQueue and Person.

Next: Build the Elevator class, which interacts with floors and uses ElevatorQueue.

Represents a floor in the building, managing people waiting and button presses.

Properties:

  • floorNumber: The number of the floor.
  • queue: A FloorQueue instance holding people waiting on this floor.
  • upButtonPressed: Boolean indicating if the up button is pressed.
  • downButtonPressed: Boolean indicating if the down button is pressed.

Methods:

  • constructor(floorNumber): Initializes the floor with the given floor number.
  • getPeopleOnFloor(): Returns the number of people on the floor.
  • addPerson(person): Adds a person to the floor's queue.
  • removePerson(person): (Not implemented in the code.)
  • pressButtons(): Updates the button states based on the destinations of people in the queue. Sets upButtonPressed if anyone wants to go up, downButtonPressed if anyone wants to go down.
Testing Ideas
  • Test constructor: let f = new Floor(0); console.assert(f.floorNumber === 0 && f.queue.isEmpty() === true);
  • Test addPerson and getPeopleOnFloor: let p = new Person('a', 0, 1); f.addPerson(p); console.assert(f.getPeopleOnFloor() === 1);
  • Test pressButtons: f.pressButtons(); console.assert(f.upButtonPressed === true && f.downButtonPressed === false);

7. Elevator Class

Why build this next: The Elevator class handles elevator movement, loading/unloading passengers, and destination logic. It depends on ElevatorQueue, Floor (for floors array), and DIRECTION constants.

Next: Build RegularElevator, a subclass of Elevator with fixed capacity.

The base class for elevators, handling movement, loading/unloading passengers, and destination logic.

Properties:

  • capacity: The maximum number of people the elevator can hold.
  • floors: An array of Floor objects representing the building's floors.
  • queue: An ElevatorQueue instance holding the people currently in the elevator.
  • currentFloor: The floor the elevator is currently on.
  • destinationFloor: The floor the elevator is heading to.

Methods:

  • constructor(capacity, floors): Initializes the elevator with the given capacity and array of floors.
  • isFull(): Returns true if the elevator has reached its capacity, false otherwise.
  • move(): Moves the elevator one floor towards its destination. Returns true if it moved, false if already at destination.
  • getNextDestinationFloor(): Determines the next floor the elevator should go to. If empty, scans floors for pressed buttons; otherwise, uses the next person's destination.
  • getNextDirection(): Returns the direction (UP or DOWN) based on the next destination floor.
  • unLoadPeople(): Removes people from the elevator who have reached their destination floor. Returns true if someone was unloaded, false otherwise.
  • loadPeople(): Loads people from the current floor into the elevator if space is available and they are going in the elevator's direction. Returns true if someone was loaded, false otherwise.
  • moveAndLoad(): Combines movement, unloading, and loading logic. If not moving, unloads and loads people, then sets the next destination.
Testing Ideas
  • Test constructor: let floors = [new Floor(0), new Floor(1)]; let e = new Elevator(4, floors); console.assert(e.capacity === 4 && e.currentFloor === 0);
  • Test isFull: console.assert(e.isFull() === false);
  • Test move: e.destinationFloor = 1; console.assert(e.move() === true && e.currentFloor === 1);
  • Test getNextDirection: e.destinationFloor = 2; console.assert(e.getNextDirection() === DIRECTION.UP);

8. RegularElevator Class

Why build this next: RegularElevator is a simple subclass of Elevator with a fixed capacity. It depends on Elevator.

Next: Build the Building class, which manages collections of floors and elevators.

A subclass of Elevator with a fixed capacity.

Properties:

  • Inherits all from Elevator, with capacity set to 4.

Methods:

  • constructor(floors): Initializes a RegularElevator with the given floors and a fixed capacity of 4.
Testing Ideas
  • Test constructor: let floors = [new Floor(0), new Floor(1)]; let re = new RegularElevator(floors); console.assert(re.capacity === 4);

9. Building Class

Why build this next: The Building class manages the overall structure, creating floors and elevators. It depends on Floor and RegularElevator.

Next: Finally, build the Simulation class, which orchestrates the entire simulation.

Manages the overall building, including floors and elevators.

Properties:

  • numElevators: Number of elevators in the building.
  • numFloors: Number of floors in the building.
  • elevators: Array of Elevator instances.
  • floors: Array of Floor instances.

Methods:

  • constructor(numElevators, numFloors): Initializes the building with the specified number of elevators and floors, creating Floor and Elevator instances.
  • moveElevatorsAndLoad(): Calls moveAndLoad on each elevator.
  • getFloor(floorNumber): Returns the Floor object for the given floor number.
  • getNumberFloors(): Returns the total number of floors.
  • generatePeopleRandomly(numPeople): Creates the specified number of people with random names, origin floors, and destination floors (ensuring origin != destination), and adds them to random floors.
Testing Ideas
  • Test constructor: let b = new Building(1, 3); console.assert(b.numElevators === 1 && b.numFloors === 3 && b.floors.length === 3 && b.elevators.length === 1);
  • Test getFloor: console.assert(b.getFloor(1).floorNumber === 1);
  • Test generatePeopleRandomly: b.generatePeopleRandomly(2); console.assert(b.floors.some(f => f.getPeopleOnFloor() > 0));

10. Simulation Class

Why build this last: The Simulation class ties everything together, managing the simulation loop, timing, and rendering. It depends on Building and all other classes.

Next: Integrate everything into the HTML file and test the simulation.

Manages the entire simulation, including timing, running/pausing, and rendering.

Properties:

  • numElevators: Number of elevators.
  • numFloors: Number of floors.
  • peoplePerSecond: Number of new people added per simulation step.
  • timer: A counter for simulation steps.
  • isRunning: Boolean indicating if the simulation is running.
  • intervalId: ID for the setInterval used for running the simulation.
  • canvas: The HTML canvas element for rendering.
  • ctx: The 2D context of the canvas.
  • building: A Building instance.

Methods:

  • constructor(numElevators, numFloors, peoplePerSecond, canvas): Initializes the simulation with the given parameters, creating a Building instance and setting up the canvas context.
  • step(): Advances the simulation by one step: generates people, presses floor buttons, moves elevators, and animates the canvas.
  • toggleRun(): Toggles between running and pausing the simulation.
  • run(): Starts the simulation by setting an interval to call step() every 1000ms.
  • pause(): Stops the simulation by clearing the interval.
  • animate(): Prints a text-based representation of the building state to the console.
  • animateCanvas(): Renders the simulation visually on the canvas, showing floors, elevators, people, and the timer.
Testing Ideas
  • Test constructor: let canvas = document.createElement('canvas'); let sim = new Simulation(1, 3, 1, canvas); console.assert(sim.numElevators === 1 && sim.building instanceof Building);
  • Test step: sim.step(); console.assert(sim.timer === 1);
  • Test toggleRun: sim.toggleRun(); console.assert(sim.isRunning === true); sim.toggleRun(); console.assert(sim.isRunning === false);

How to Recreate the Project

  1. Implement Classes in Order: Follow the step-by-step class implementation above (1-10), building each class in the specified order to manage dependencies.
  2. HTML Setup: Create elevator.html with a canvas element and a button to start/pause the simulation. Include scripts to import and run the Simulation class.
  3. Testing: Run the simulation and verify that elevators move correctly, people are loaded/unloaded, and the canvas updates properly.

This summary covers all the essential components. The project uses ES6 modules, so ensure your environment supports them (e.g., via a local server). For visualization, the canvas drawing logic can be adjusted for different layouts.
